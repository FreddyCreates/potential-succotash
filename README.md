# Front-End Is All Intelligence

A sovereign frontend model civilization — 150 AI models derived from real frontend technologies, reinterpreted as sovereign entities that perform real work, capture value, earn tokens, and express themselves through three faces.

## What This Is

This is not a list of frontend tools. This is an **economic architecture** and **model civilization** where every frontend technology becomes a sovereign intelligence with:

- **3 sovereign model variants** (reinterpretations beyond default use)
- **2 sub-forms** (specialized branches)
- **1 alpha multimodal coordination form** (cross-variant orchestrator)
- **3 engines** (from 6 discovery engine types)
- **3 faces** (internal organism / external product / public expression)
- **Token economy participation** (WRK / DSC / SVN three-tier system)
- **Value-capture integration** (proof-based ledger routing)

## Architecture Components

### A. Value-Capture Architecture
How models discover value, prove their work, and route it into the ledger. Six discovery engine types (Pattern, Optimization, Creation, Routing, Analysis, Integration), five proof types, and contract settlement.
→ See [`architecture/value-capture-architecture.md`](architecture/value-capture-architecture.md)

### B. Token Economy
Three-tier token system: **WRK** (base work token), **DSC** (discovery token), **SVN** (sovereign token). Complete with issuance rules, burn/lock/sink mechanics, conversion paths, and governance.
→ See [`architecture/token-economy-architecture.md`](architecture/token-economy-architecture.md)

### C. Model-Face Split
Every model expresses through three faces: **Internal** (organism function), **External** (product/service function), **Facing** (public expression). Includes coordination protocol and evolution stages.
→ See [`architecture/model-face-split.md`](architecture/model-face-split.md)

### D. 150-Model Registry
The complete registry of all 150 models in CSV format (openable in Excel). Each row contains all 21 attributes per model.
→ See [`registry/frontend-sovereign-registry.csv`](registry/frontend-sovereign-registry.csv)

## Registry Columns

The CSV registry contains 21 columns per model:

| Column | Description |
|---|---|
| ID | Unique identifier (1-150) |
| Source_Technology | Original frontend tech name |
| Category | Technology category |
| Sovereign_Model_1/2/3 | Three sovereign reinterpretations |
| Sub_Form_1/2 | Two sub-branches |
| Alpha_Multimodal_Form | Cross-variant coordination form |
| Engine_1/2/3 | Three assigned discovery engines |
| Original_Default_Use | What the tech normally does |
| Sovereign_Use_1/2/3 | Three flipped sovereign capabilities |
| Internal_Use | Organism function |
| External_Use | Product/service function |
| Facing_Use | Public expression |
| Organism_Placement | Body system mapping |
| Ledger_Relevance | Token economy connection |
