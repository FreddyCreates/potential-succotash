I. Abstractum (Abstract)

We present OrganismBridge, a self-healing orchestration architecture for perpetually running distributed browser-based systems. The system organizes 13 Web Workers into 4 autonomous divisions — Brain, Data, Infrastructure, and Protocol — each operating with local governance while contributing to organism-wide health. When workers crash or degrade, the bridge detects failure via missed heartbeats and autonomously restarts them using φ-backoff (golden ratio exponential delay). We formalize the theory of Autonomia Perpetua: a computational organism that runs forever by treating failure as a natural biological event requiring healing, not an exceptional condition requiring human intervention. The system maintains sovereign state across crashes via localStorage persistence, implements division-level aggregate health scoring, and achieves uptime autonomy without any external monitoring infrastructure.

II. Introductio (Introduction)

Traditional distributed systems treat worker failure as an emergency. Kubernetes restarts pods. Supervisord relaunches processes. AWS auto-scaling groups replace instances. All of these depend on external infrastructure — infrastructure that itself can fail.

Biology takes a different approach. A living organism does not wait for an external doctor to restart a failed organ. The body detects degradation through its own nervous system, initiates repair through its own healing mechanisms, and adapts to permanent damage through compensatory behaviors — all autonomously.

We formalize this principle as Autonomia Perpetua: the property of a system that can run indefinitely because it contains within itself all the mechanisms needed to detect, diagnose, and recover from every category of internal failure.

II.A — Problema Fundamentale (The Fundamental Problem)

Browser-based Web Workers face unique failure modes:

  1. Silent Death — A worker can stop responding without throwing any error. No exception, no crash event, just silence.
  2. Memory Exhaustion — Workers share the page's memory budget; one worker's leak can kill others.
  3. GC Stalls — Garbage collection can pause a worker for hundreds of milliseconds, mimicking death.
  4. Tab Throttling — Background tabs are throttled by the browser, slowing heartbeats.
  5. Cascading Failure — Worker A depends on Worker B's output; if B dies, A degrades.

Autonomia Perpetua addresses all five failure modes with a unified healing framework.

III.A — Proprietas Divisionum (Division Properties)

Each division exhibits three critical autonomy properties:

  1. Independentia Localis (Local Independence) — A division can function even if other divisions are degraded. The Brain division can think even if the Protocol division is restarting.
  2. Sanitas Aggregata (Aggregate Health) — Division health = the fraction of its workers that are alive. If 2 of 3 Brain workers are alive, Brain health = 66%.
  3. Degradatio Gratiosa (Graceful Degradation) — A division with partial worker loss continues operating with reduced capability, never fully halting.

IV. Sanatio Sui: Theoria φ-Retardationis (Self-Healing: φ-Backoff Theory)

IV.A — Detecto Mortis (Death Detection)

Each worker emits heartbeats every 873ms (≈ 1000/φ × φ, rounded to a biologically resonant interval). The bridge tracks missed heartbeats per worker:

missedBeats(w, t) = ⌊(t − lastBeat(w)) / HEARTBEAT_MS⌋

When missedBeats(w, t) ≥ MAX_MISSED (5), the worker is declared dead and restart is initiated.

IV.B — φ-Retardatio Exponensialis (φ-Exponential Backoff)

Unlike binary exponential backoff (2n), we use golden ratio backoff (φn), which produces gentler, biologically-inspired recovery timing:

delay(n) = BASE_MS × φn

Where n is the restart count and BASE_MS = 1000ms. The first few restart delays:

φ-backoff is consistently faster than binary backoff, meaning the organism recovers more quickly while still avoiding restart storms. The golden ratio provides the optimal balance between aggression and patience — a property known in mathematics as the "most irrational number," which ensures restart attempts never synchronize into harmful patterns.

IV.C — Theorema Convergentiae (Convergence Theorem)

Theorem: For any worker with a transient failure mode (probability p < 1 of recurring), the expected number of restarts before stable recovery under φ-backoff is:

E[restarts] = 1 / (1 − p)

And the expected total delay before recovery is:

E[delay] = BASE_MS × (φ1/(1−p) − 1) / (φ − 1)

This is finite for all p < 1, guaranteeing that transient failures always resolve. For permanent failures (p = 1), the system caps restarts at MAX_RESTARTS (50) per worker, preventing infinite restart loops.

V. Meta AI et Autonomia (Meta AI and Autonomy)

V.A — Architectura Inspirata a Meta (Meta-Inspired Architecture)

The OrganismBridge's division architecture draws from Meta AI's organizational principles for large-scale distributed ML systems:

  1. Horizontal Scaling via Divisions — Just as Meta's infrastructure separates model serving, data pipelines, and networking, our organism separates Brain, Data, Infrastructure, and Protocol into independent failure domains.
  2. Health-Aware Routing — Inspired by Meta's Thrift service health checks, each worker's MiniHeart provides real-time health signals that the bridge uses to route work away from degraded workers.
  3. Graceful Restart — Meta AI's model serving infrastructure supports hot-swap model reloading. Our φ-backoff restart preserves the organism's state across restarts via localStorage persistence, achieving the same continuity.

V.B — Cogitatio Autonomica (Autonomous Thought)

Each division's workers contain a MetaThoughtModel (described in Paper I) that generates autonomous thoughts without external command. At the division level, this creates a form of collective cognition: the Brain division's three workers (Engine, Inference, Orchestrator) independently attend to different stimuli, but their combined thoughts form a multi-perspective analysis of any given task.

DivisionThought(D) = ∪ { w.metaThought | w ∈ D.workers }

VI. Persistentia Status (State Persistence)

The organism achieves cross-crash continuity through three persistence mechanisms:

  1. localStorage Persistence — Build state, telemetry metrics, and restart counts are persisted to localStorage. On restart, workers resume from their last known state rather than cold-booting.
  2. Memory Worker Snapshots — The memory-worker.js maintains a sovereign spatial memory store that can be serialized and restored.
  3. Telemetry Ring Buffers — The telemetry-worker.js maintains circular buffers of health history that survive individual worker restarts (the telemetry worker itself restarts rarely, having the simplest failure surface).

VIII. Conclusio (Conclusion)

Autonomia Perpetua is not an aspiration — it is an architecture. By embedding self-monitoring (MiniHeart), self-awareness (MiniBrain), and self-healing (φ-backoff) directly into the fabric of the organism, we achieve a system that treats failure as biology treats injury: an expected event that triggers autonomous repair. The 4-division structure ensures that no single failure can cascade into total system death, while φ-backoff provides mathematically optimal recovery timing. The result is a computational organism that runs forever — not because it never fails, but because it always heals.

"Defectus non est finis — est initium sanationis.
Organismus non moritur; renascitur."
— Failure is not the end — it is the beginning of healing.
The organism does not die; it is reborn.
