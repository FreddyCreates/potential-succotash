Cloudflare Workers Deployment

Deploy the organism to Cloudflare's global edge network for sub-50ms latency worldwide.

⚠️ Prerequisites

You'll need a Cloudflare account with Workers, D1, KV, R2, Vectorize, and Queues enabled.

Worker Configuration

Each worker has a wrangler.toml configuration:

name = "api-node" compatibility_date = "2024-01-01" [ai] binding = "AI" [[kv_namespaces]] binding = "CACHE" id = "your-kv-id" [[d1_databases]] binding = "DB" database_id = "your-d1-id" [[vectorize]] binding = "INDEX" index_name = "medinatech-index"

Deployment Script

#!/bin/bash # scripts/deploy-all-workers.sh WORKERS=( "api-node" "coordinator" "gate-node" "knowledge-realm" "nova-sovereign" "cache-cognition" "probe-node" "honeypot-admin" "honeypot-portal" ) for worker in "${WORKERS[@]}"; do echo "Deploying $worker..." npx wrangler deploy --config workers/$worker/wrangler.toml done
